# Emby-In-One Update Plan

当前稳定版本：**V1.2.1**

---

## 🚀 V1.3（Pre-release）：后端重构为 Go

### 目标

将当前后端从 Node.js 重构为 Go，为后续的多用户管理、权限控制、播放前检查和更高并发场景打下性能基础。

### 版本定位

V1.3 的重点是 **后端架构升级**，不是直接交付多用户能力。  
核心目标是先把后端性能、并发处理能力、可维护性和后续扩展空间提升起来。

### 计划内容

- 将主要后端服务逐步迁移到 Go
- 保持现有 Emby 客户端访问方式尽量兼容
- 保留多服务器聚合、搜索去重、播放代理、日志、管理面板等核心能力
- 对目前较为混乱的 UA 伪装逻辑进行精简与整理，统一行为和配置方式
- 重新整理后端模块边界，为后续多用户管理预留结构
- 为后续按用户维度做权限判断、服务器筛选、播放限制预检查做基础设施准备

### 预期结果

- 获得一个性能更高、并发处理更稳定的 Go 后端版本
- 为 V1.4 的多用户管理提供更合适的技术基础

### 不在 V1.3 范围内

- 不在本版本正式交付完整多用户管理功能
- 不在本版本正式交付按用户分配服务器权限
- 不在本版本正式交付同时播放人数限制控制

> 已于 2026-03-30 发布 Pre-release 版本。

## V1.4 计划：多用户管理与播放限制

### 目标

在 Go 后端基础上实现完整的多用户管理能力，并支持按用户控制可访问的上游 Emby 服务器，以及播放前的同时观看人数限制检查。

### 计划内容

#### 1. 多用户管理

- 支持创建和管理多个 Emby In One 用户
- 支持为不同用户设置独立访问权限
- 支持在管理面板中维护用户信息

#### 2. 按用户分配可访问服务器

- 支持为每个用户配置可请求到的 Emby 服务器范围
- 用户发起请求时，仅能访问被授权的上游服务器
- 为后续更细粒度的权限控制预留扩展空间

#### 3. 同时播放管理

- 支持为部分上游 Emby 服务器设置“同时观看人数限制”
- Emby In One 在向对应服务器发起播放请求前，先检查当前正在观看该服务器的 Emby In One 用户数量
- 如果当前人数 **大于或等于限制值**，则不向上游请求播放
- 直接返回“同时观看已达上限”之类的限制提示

### 预期结果

- Emby In One 从单管理员入口升级为可用的多用户聚合平台
- 可以按用户精细控制其能访问哪些上游服务器
- 可以在代理层提前拦截超出并发限制的播放请求，减少上游服务器压力和风险

## 版本关系

- **V1.2**：全面安全审计修复（API 脱敏、CORS 加固、输入校验、密码安全）；Passthrough 透传修复（五级优先级链、客户端身份持久化、per-server headers）；超时日志增强；管理员密码重置机制；上游 HTML 错误拦截；Graceful shutdown 资源清理
- **V1.3**：完成 Go 后端重构，性能与并发处理能力大幅提升，为多用户能力做架构铺垫
- **V1.4**：实现多用户管理、用户可访问服务器控制、同时播放管理
